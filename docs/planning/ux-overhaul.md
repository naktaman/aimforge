# AimForge UI/UX 전면 개편 설계서

> **작성일**: 2026-04-07
> **원칙**: "감도를 깎으러 온 사람" — 앱을 열면 감도 최적화가 메인이어야 한다.
> **비주얼**: Cold Forge 디자인 시스템 (docs/planning/cold-forge-ui-redesign.md)

## 현재 문제 진단

### A. 설정 화면 (DisplaySettings.tsx)
**유저 피드백**: "설정창도 이따위로 부실한건 살면서처음봣어"

현재 설정:
- 디스플레이: 화면모드, FPS제한, VSync, 렌더스케일 — **이게 전부**
- 언어: 한국어/영어 셀렉트 1개
- 사운드: ON/OFF 토글 1개
- 데이터: JSON 내보내기 버튼 1개
- 앱 정보: 버전 텍스트 3줄

**빠진 것들** (사용자가 설정 화면에서 기대하는 것):
1. **하드웨어 설정**: DPI, 폴링레이트, 마우스 모델 (현재 대시보드 상단바에 분산)
2. **게임 감도 설정**: 현재 활성 프로필의 감도/FOV 편집 (대시보드에 있지만 설정에도 있어야 함)
3. **게임 프로필 바로가기**: 프로필 관리 진입점
4. **입력 설정**: 마우스 가속 체크, raw input 토글, 감도 단위 선택 (cm/360, inch/360, eDPI)
5. **3D 렌더링 상세**: 안티앨리어싱, 텍스처 품질, 파티클 밀도
6. **크로스헤어**: 현재 탭에 있지만 설정에서도 접근 가능해야
7. **알림/HUD**: 인게임 HUD 표시 옵션, 히트 피드백 스타일
8. **단축키**: 키바인드 설정
9. **데이터 상세**: 내보내기 + 가져오기 + 초기화 + 자동백업 주기
10. **계정**: Steam 연동 상태

### B. 메인 대시보드 (ScenarioSelect → SensitivityTab)
**유저 피드백**: "어드밴스드 진입 후 뭘 해야할지 모르겠음"

현재 구조:
- 상단바: 탭 3개 (감도프로파일 | 훈련 | 분석) + 게임/DPI/감도 입력
- 감도탭: 좌(감도카드) | 중앙(캘리브레이션) | 우(게임프로필/변환 도구)
- 훈련탭: 시나리오 직접 파라미터 입력 (유저에게 시간/갯수 입력 요구)
- 분석탭: 빈 차트 + empty state만

**문제**:
1. 온보딩 끝나고 메인에 들어왔을 때 "다음 할 일"이 명확하지 않음
2. 캘리브레이션 안 했으면 → "최적 감도 찾기" 안내가 더 강해야
3. DNA 배터리 안 했으면 → "능력 측정" 안내
4. 빈 화면투성이 — 데이터 없을 때 뭘 해야 하는지 안내가 약함
5. 감도 입력이 대시보드 상단바에 있는데 설정에도 있어야

### C. 메뉴 구조
**유저 피드백**: "크로스헤어가 트레이닝과 같은 라인에", "Tools에 FOV 비교 방치"

현재:
- Simple 모드: 기능 대폭 숨김
- Advanced 모드: 전부 표시인데 정리 안 됨
- 크로스헤어가 탭 옆에 + 아이콘으로 숨어있음

**개선 방향**:
- 메인 플로우 (감도 최적화) vs 도구 (부가기능) 명확 분리
- 크로스헤어는 설정에 포함
- Tools 안의 기능들은 맥락에 따라 노출 (DNA 결과 나온 후에 활성화)

### D. 빈 화면/콘텐츠 부족
- 분석탭 전체가 empty state
- 세션 히스토리: 빈 화면
- 프로그레스 대시보드: 빈 화면
- 트레이닝 처방: DNA 없으면 빈 화면

**개선**: "아직 데이터가 없습니다" → "이걸 먼저 해보세요: [캘리브레이션 시작]" 같은 actionable empty state

## 개편 태스크 목록

### UX-1: 설정 화면 충실화

**현재**: 5개 섹션 (디스플레이, 언어, 사운드, 데이터, 정보) — 내용물 빈약

**개편 후 섹션 구조**:

```
설정
├── 하드웨어
│   ├── DPI: [800]
│   ├── 폴링레이트: [1000] Hz
│   ├── 마우스 가속: ☐ 비활성화 (권장)
│   └── Raw Input: ☑ 활성화
│
├── 감도 & 프로필
│   ├── 현재 활성 프로필: [CS2 — 감도 1.0, 46.2 cm/360]
│   ├── 감도 단위: [cm/360 | inch/360 | eDPI]
│   └── [게임 프로필 관리 →]
│
├── 디스플레이
│   ├── 화면 모드: [창모드 | 보더리스 | 전체화면]
│   ├── FPS 제한: [무제한 | 60 | 144 | 240 | ...]
│   ├── VSync: ☐
│   ├── 렌더 스케일: [100%] (슬라이더 50-200)
│   ├── 안티앨리어싱: [없음 | FXAA | MSAA 2x | 4x]
│   └── 렌더 품질: [낮음 | 보통 | 높음]
│
├── 크로스헤어
│   └── [크로스헤어 편집기 열기 →] (현재 CrosshairSettings 재활용)
│
├── 사운드
│   ├── 마스터 볼륨: [80%] (슬라이더)
│   ├── 효과음: ☑
│   ├── 히트 사운드: ☑
│   └── 히트 사운드 종류: [기본 | 게임1 | 게임2]

│   ├── 인게임 FPS 표시: ☑
│   ├── 히트마커 표시: ☑
│   ├── 피격 피드백: [없음 | 색상 | 소리 | 둘 다]
│   └── 점수 팝업: ☑
│
├── 언어
│   └── 언어: [한국어 | English]
│
├── 데이터 관리
│   ├── [데이터 내보내기 (JSON)]

│   └── 자동 백업: [비활성 | 매일 | 매주]

    ├── 버전: v0.2.0
```

### UX-2: 메인 대시보드 — 감도 최적화 메인 플로우

**현재 문제**: 처음 들어오면 뭘 해야 할지 모름

**개편 방향**:
1. 온보딩 직후 → "다음 단계" 안내 카드 (캘리브레이션 시작)
2. 캘리브레이션 완료 후 → "Aim DNA 측정" 안내
3. DNA 완료 후 → "훈련 추천" 안내 + 인사이트
4. 모든 단계 완료 → 데이터 기반 대시보드 (지금의 빈 차트 대신 실 데이터)

구체적으로:
- SensitivityTab 중앙 히어로 영역: 상태에 따라 다른 CTA
  - 프로필 없음 → "게임 프로필 생성"
  - 캘리브레이션 안 함 → "최적 감도 찾기" (현재와 동일하지만 더 강조)
  - 캘리브레이션 완료 → "결과: XX cm/360 적용됨" + "DNA 측정 시작"
  - DNA 완료 → 6축 레이더 미니 차트 + "약점 훈련 시작"

---

### UX-3: 메뉴/네비게이션 정리

- 탭명: "감도 프로파일" → "감도 최적화" (더 명확)
- "훈련" 탭: 프리셋 먼저, 커스텀은 접힌 상태
- "분석" 탭: 데이터 없으면 "캘리브레이션/DNA 먼저 하세요" 안내 + 빠른 시작 버튼
- 크로스헤어: 설정으로 이동 + 탭에서 제거
- 하단 카테고리 카드 3개: 현재 상태 기반 동적 변경

---

### UX-4: 빈 화면 → Actionable Empty State

모든 빈 화면에:
1. 왜 비어있는지 설명 (데이터가 없어서)
2. 뭘 하면 채워지는지 안내
3. 바로 실행할 수 있는 버튼

---

## 실행 순서

1. **UX-1: 설정 화면 충실화** — 가장 명확하고 독립적
2. **UX-2: 메인 대시보드 개편** — 상태 기반 CTA
3. **UX-3: 메뉴 정리** — 탭명, 크로스헤어 위치 등
4. **UX-4: 빈 화면 개선** — 모든 empty state 강화

UX-1부터 코드 세션에 위임.
